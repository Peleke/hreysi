## The Journey

Today started as recon, not a build. The ask was fuzzy on purpose — "we had some
notes, the learning machinery doesn't need to work, we just need the slice that
documents work." The notes turned out to be real: cadence-planner's
`buildlog-autofire-plan.md` was basically a spec, and it named the exact failure
we'd been living with — buildlog's capture logic is *welded inside* `buildlog
commit`, so the git `post-commit` hook can only nudge, never capture, and the
`pre-commit` wall pushed every commit through a `BUILDLOG_COMMIT=1` bypass that
runs raw git and skips the capture entirely. Friction up, value zero.

The decision that unlocked everything was to stop treating this as a buildlog
patch and extract the capture slice into its own tool. Once we said "the learning
loop can die," the whole thing collapsed to something tiny: git commit → append
to a dated file. We talked Go vs Rust and I initially over-justified Rust on
"blazing speed" — Peleke corrected the framing cleanly: with qortex/PPR out of
scope there's no compute-heavy path, so the only real reason to compile is
**single-binary, zero-dependency distribution**. Go, decisively. Named it
`hreysi` (Old Norse for cairn) to keep the Norse thread.

The build itself was smooth — stdlib-only, real git repos in the tests (no mocks,
which is the only kind of test that proves capture actually works). The friction
was all in the *distribution* loop, which is fitting, because proving the loop was
the whole point:

- **The irony moment:** my very first attempt to commit hreysi was *blocked by the
  global buildlog enforce hook* — the exact "wall + bypass" the autofire plan says
  to tear down, firing live in my face. hreysi's own post-commit hook is decoupled
  from that entirely, so committing through the bypass still captured the commit.
  That wasn't a demo I staged; it just happened, and it's the best possible
  argument for the redesign.
- **The silent failure:** `gh repo create --push` died with "Could not read from
  remote repository." The repo *had* been created — the SSH key just wasn't loaded
  in this shell. Switched the remote to HTTPS with the gh token as credential
  helper and it went through. Classic "failed silently in a way that looks like a
  bigger problem than it is."

Then the loop closed for real: tagged `v0.1.0`, GitHub Actions + goreleaser built
cross-platform tarballs, and `install.sh` pulled the darwin_arm64 build back down
and ran it — install from a pipeline-published artifact, not a local build.
Homebrew came next: created `Peleke/homebrew-tap`, wired goreleaser's `brews`
block, cut `v0.1.1`, and `brew install Peleke/tap/hreysi` landed the binary in
`/opt/homebrew/bin`. Every layer real, every layer proven.

The last turn was the sharpest design conversation: commit data doesn't *expand*
into a buildlog on its own. The spine is just an index — the lived experience
(the dead ends above, the reasoning) lives in the session and evaporates when it
ends. That's this section. This entry is the proof: hreysi wrote the `## Commits`
spine mechanically, and the `expand` skill grafted the story on top from what we
actually did — which is exactly what will feed linwheel.

## Improvements

### Architectural
- Capture must be a side-effect of committing, not a separate command — move it to
  a non-blocking `post-commit` hook so it fires no matter how you commit.
- Make the output directory the decoupling boundary: the producer only writes, all
  consumers only read. Section ownership within a file (`## Commits` vs
  `## The Journey`) lets mechanical and agentic writers coexist without collision.
- Commit metadata is a spine, not a body. Narrative expansion is a *join* of spine
  and lived experience; it can't be derived from git alone.

### Tooling
- Pick a compiled language for *distribution* (single static binary), not for
  speed, when the workload is I/O-bound orchestration. Name the real reason.
- Embed shipped assets (skills) in the Go binary with `//go:embed` and drop them on
  `init` — capture and expansion ship together, still decoupled at runtime.

### Process
- Prove the release loop end-to-end (tag → CI → published artifact → install from
  that artifact), not just a green build. Installing from the package source is the
  only thing that proves "pipx/brew/curl → it works."
- Expand narrative while the session context is warm; the texture (dead ends, the
  aha) decays fast once the conversation is gone.

### Gotchas
- `gh repo create --push` fails on "Could not read from remote repository" when the
  SSH key isn't loaded — the repo may already exist; switch to HTTPS + `gh auth
  setup-git`.
- The built-in Actions `GITHUB_TOKEN` can't push to a *different* repo, so a
  Homebrew tap needs a separate token secret (rotate the broad gh token to a
  fine-grained PAT scoped to the tap).
